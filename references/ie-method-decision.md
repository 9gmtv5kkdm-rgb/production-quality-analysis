# IE 七大手法 → 问题映射决策表（能力层 · 怎么用）

> SKILL.md「步骤4」给了手法与问题的对应，本表补**每一手的落地动作**：何时用、具体怎么做、产出什么。
> 原则：每个被帕累托锁定的关键少数问题，至少配 1 个 IE 手法 + 鱼骨图 + 5Why。

## 一、七大手法落地决策表

| IE 手法 | 适用问题信号 | 具体落地动作（可执行） | 产出 |
|---------|--------------|------------------------|------|
| **流程程序分析** | 尺寸超差(A)/变形(D)/材质(H)；存在返修回路 | 画 下料→加工→装配→检验→返修 流程，标注每环节增值/非增值，找返修回路与检验缺失点 | 流程现状图 + 瓶颈环节清单 |
| **动素分析(18动素)** | 表面缺陷(E)/修磨(F)；碰伤、过磨 | 分解上下料/修磨动作，标出「拿取/对准/持住/迟延」等浪费动素，定位碰伤与过磨发生点 | 动素分解表 + 浪费动素清单 |
| **双手作业分析** | E/F/G；双手不对称、单手扶件失稳 | 画左右手动作同步图，找单手空闲/双手干涉，定位失稳导致的偏移、错边 | 双手作业图 + 不平衡点 |
| **人机配合分析** | A/B/G；找正/测量/夹紧对尺寸影响 | 画 人-机 时间线，标设备等待、人干预点，定位同步性偏差来源 | 人机配合图 + 干涉点 |
| **工作抽样分析** | B/数据治理；缺陷集中节点不明 | 设定观测周期与频次，抽样记录缺陷发生时段（换活/连续运行/录入缺失），定位高发节点 | 抽样观测表 + 高发时段 |
| **动作经济原则(22)** | E/F；碰撞、过磨 | 逐条核对「缩短距离/减少动作数/利用重力/定置工具」等原则，给减碰减磨的工位改善 | 违反原则清单 + 改善点 |
| **标准工时分析** | A/F；节拍压缩、凭手感停 | 测算试模/修磨标准工时，对比实际，识别因抢节拍导致的尺寸/修磨失控 | 标准工时 vs 实际对比 |

## 三、鱼骨图 + 5Why 强制结构

每个关键问题必须输出：
1. **鱼骨图**：6 维度（人/机/料/法/环/测）各列 ≥1 条末端原因；维度按「族主导维度」优先展开，但其余维度不得空（至少写「暂未发现」）。
2. **5Why 表**：连续追问至末层标「根本原因」，且末层必须落到「可采取措施」的层面（不能停在「操作人员疏忽」这类不可改结论）。
3. **结构化结果表**：`根因 | 影响因素 | 改进方向` 三者并列。

## 四、边界与异常处理

- **仅 1 条记录 / 超小样本**：不做帕累托占比判定，改为「逐条深挖 + 同类经验库对标」，结论标注「样本量 N=1，结论为方向性」。
- **工序全缺失**：跳过主责工序排名，把「工序字段缺失」本身作为数据治理发现，按工作抽样思路给补录建议。
- **某族占比 100%**：复核是否关键词过宽吞并了其它族（见 defect-taxonomy 自检）。
- **无明确根因**：末层 5Why 写「当前数据无法定位，建议补充：XX 记录/现场复现」，禁止编造。
